# Contributing

Thank you for contributing! Please follow these rules:

## Branches

- Work in `feature/<short-description>` branches.
- Keep feature branches small and focused.

## Commit messages

Use a simple convention:

- `feat:` new feature
- `fix:` bug fix
- `docs:` documentation
- `chore:` maintenance
- `refactor:` code changes without feature or fix

## Pull Requests

- Target `dev` branch for feature PRs.
- Provide a short description, list changes, and link associated tasks.
- Use the PR template provided.
